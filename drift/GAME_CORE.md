# 游戏核心 · DRIFT

## 目标（Goal）

**找出并验证「谁在改写世界规则」** — 在规则再次漂移前，抵达接口并确认 REQUEST 真伪。

## 代价（Tradeoff）

| 选 A | 失去 B |
|------|--------|
| 深探节点 C（耗 memory） | 可能赶不上节点 B 谜题 |
| 跟错标牌求快 | 循环重置，记忆崩散 |
| 在门前久候不进门 | 知引擎在门外，但进度停滞 |

## 循环（Loop）

每个探索节拍：

1. **移动** → 2. **交互物体/终端** → 3. **发现规则与昨日不一致** → 4. **记录或验证** → 5. **章节/场景自然收束**

**去日志化：** 不用 `进度：`/`存档：`；见 [PLAYER_UX.md](PLAYER_UX.md)。

## 限制（Constraint）

- 交互消耗 `memory` / `attention`（规则会变）
- 节点 B 谜题必须解对，否则 loop 结局

## 爽点

你无法确定：是世界坏了，还是**你在被测试**。
