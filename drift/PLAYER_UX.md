# 输出格式 · DRIFT（去日志化）

> **主人只看悬疑叙事**，不看机器进度条。  
> `events.jsonl` / `state.json` 仅引擎后台，**禁止粘贴给主人**。

---

## 原则

| 做 | 不做 |
|----|------|
| 终端 `RULE SET`、报错、〔终端〕播报 | `进度：` `存档：` 管道行 |
| 章节标题 `## 第N章 · …` 作节拍分界 | `｜规则v2｜线索0｜节拍3` |
| 线索写进发现过程（「我记下…」） | `**线索：**`、线索 3/3 |
| 收束用文学/悬疑尾声 | `结局·interface` 式 ID |

**终端输出是戏内文字，不是日志。**

---

## 推进 = 章节 + 场景

```
## 第3章 · 规则第二次漂移

我走向终端。屏幕亮起：

`RULE SET v2: interaction consumes memory`

……
```

下一段用新章节或新场景自然衔接 —— **无需**另起 `进度：` 行。

---

## 收束（第一章 / 第二章钩子）

用 prose 结束，例如：

> 门缝里的光不再闪烁。终端最后一行还停在 `REQUEST: RULE PATCH`。我不知道门外是谁，但我知道规则不会再信我了。

第二章钩子同理 —— 用裂隙、回响、冷重放收束，不用 `存档：第二章钩子尽·…`。

续玩：主人说「继续」→ 从下一场景/章节接写。
